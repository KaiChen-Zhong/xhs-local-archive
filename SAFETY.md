# 安全边界

本项目只整理当前浏览器登录态下已经可见或页面正常返回的收藏/点赞卡片数据。项目不包含反检测、绕过、破解或批量自动化攻击能力。

## 已实现约束

- 用户主动点击后才开始采集或扫描。
- 页面加载时不做后台持久采集。
- 不自动打开帖子；只有用户点击已采集卡片封面时才打开对应小红书页面。
- 不做详情页 hydration。
- 不读取帖子正文。
- 不采集评论。
- 分类只使用卡片标题和封面。
- `fetch` / `XMLHttpRequest` 镜像只在用户触发采集/扫描后启用。
- 网络镜像限定到小红书收藏、点赞、feed、搜索、卡片相关接口路径。
- 网络镜像可在停止、风险事件后暂停。
- 扫描滚动步长、等待时间、单次发现数量都有保守限制。
- 扫描启动有冷却时间。
- 批量导出每次最多处理 10 条，并做节奏控制。
- 检测到登录、验证、二维码、访问受限或频繁访问提示后停止扫描。
- 访问受限或验证触发后，采集动作锁定 15 分钟。
- 不自动删除或修改小红书账号中的任何内容。
- 本地删除只影响本地数据库和本地导出文件。
- Native Host 只写入本地归档目录。
- 媒体下载有超时、大小限制，并拒绝 localhost/private network 地址。
- 本地媒体读取必须通过 Native Host 路径校验，且文件必须位于归档目录内。
- AI API key 不会通过设置接口明文返回；Windows 使用 DPAPI，其他平台使用本地随机密钥加密。

## 支持的数据来源

- 当前页面 DOM 中用户可见的收藏/点赞卡片。
- 当前页面已返回的列表、feed、收藏、点赞、搜索相关网络响应。
- 卡片标题、封面、作者、noteId、来源 URL。

## 不支持也不承诺

- 验证码绕过。
- 指纹伪装。
- 请求签名破解。
- 隐藏浏览器自动化。
- 后台打开帖子补全文本。
- 抓取帖子正文或评论。
- 读取当前账号无权访问、已删除、私密或受限内容。
- 保证小红书原始链接永久可打开。
- 保证任意快速滚动都不会漏掉未加载卡片。

## 使用建议

- 保持低频采集。
- 遇到验证或访问受限后停止操作，按正常浏览器流程处理。
- 把本项目视作本地收藏整理工具，而不是数据抓取或平台绕过工具。
